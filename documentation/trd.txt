1. SYSTEM OVERVIEW
================================================================================

SupportEnv is a FastAPI-based reinforcement learning server that exposes a
customer-support simulation through REST, WebSocket, and Gradio interfaces.
It conforms to the OpenEnv specification (openenv-core >= 0.1.0), allowing any
compatible RL client to train and evaluate agents.

High-level data flow:

    ┌─────────┐   HTTP / WS    ┌──────────────┐
    │  Client  │ ──────────────▶│  FastAPI App  │
    │ (agent)  │ ◀──────────────│  (server/)    │
    └─────────┘   JSON          └──────┬───────┘
                                       │
                        ┌──────────────┼──────────────┐
                        ▼              ▼              ▼
                  ┌──────────┐  ┌───────────┐  ┌──────────┐
                  │Env Logic │  │  Grader   │  │  Reward  │
                  │  (RL)    │  │(scoring)  │  │ Engine   │
                  └──────────┘  └───────────┘  └──────────┘
                        │
                        ▼
                  ┌──────────────┐
                  │Ticket Gen.   │
                  │(templates)   │
                  └──────────────┘

4.1 Core Classes & Inheritance

    openenv.core.env_server.Environment   (abstract base)
        └── SupportEnvironment            (server/environment.py)

    openenv.core.env_server.Action        (abstract base)
        └── SupportAction                 (models.py)

        └── SupportState                  (models.py)

    openenv.core.env_client.EnvClient     (generic client)
        └── SupportEnv (client)           (client.py)

4.2 SupportEnvironment — Lifecycle

    ┌────────────────────────────────────────────────────────────────┐
    │                      EPISODE LIFECYCLE                         │
    ├────────────────────────────────────────────────────────────────┤
    │                                                                │
    │  reset(seed, difficulty, task_id)                              │
    │    ├─ Set random seed                                         │
    │    ├─ TicketGenerator.generate_ticket(difficulty)              │
    │    ├─ Initialise SupportState                                 │
    │    ├─ Reset RewardEngine                                      │
    │    └─ Return initial SupportObservation (done=False)          │
    │                                                                │
    │  step(action) — called in loop until done==True               │
    │    ├─ Increment step_count                                    │
    │    ├─ Record action in action_history                         │
    │    ├─ Dispatch _handle_{classify|respond|escalate|...}        │
    │    ├─ RewardEngine.compute_reward(...)                        │
    │    ├─ Check termination conditions                            │
    │    │    ├─ Resolved?                                          │
    │    │    ├─ Escalated?                                         │
    │    │    └─ Max steps reached?                                 │
    │    ├─ (If done) compute_episode_final_reward(...)             │
    │    └─ Return SupportObservation (reward, done, message)       │
    │                                                                │
    │  grade_episode()                                              │
    │    ├─ Collect episode_data (actions, targets, flags)          │
    │    └─ SupportGrader.grade_episode(**data) → GradeResult       │
    │         ├─ score:     float (0.0, 1.0) strictly                 │
    │         ├─ breakdown: Dict[str, float]                        │
    │         ├─ feedback:  str                                     │
    │         └─ passed:    bool                                    │
    │                                                                │
    └────────────────────────────────────────────────────────────────┘

4.3 Ticket Generator (server/ticket_generator.py — 15 KB)

    - Maintains TASK_DEFINITIONS dict keyed by difficulty.
    - Each definition includes:
        task_id, name, description, max_steps,
        ticket templates (subject, body, category, sentiment,
        expected_resolution, requires_escalation).
    - TicketGenerator(seed) → generate_ticket(difficulty, task_id)
      Returns a dict with all ticket fields.

4.4 Reward Engine (server/reward.py — 11 KB)

    - compute_reward():  Per-step reward for individual actions.
    - compute_episode_final_reward():  End-of-episode bonus / penalty.
    - RewardBreakdown dataclass holds per-dimension scores.
    - All computation is deterministic (no LLM calls).

4.5 Grader (server/graders.py — 13 KB)

    - SupportGrader.grade_episode() is the single entry point.
    - Dimension weights are read from TASK_DEFINITIONS or openenv.yaml.
    - GradeResult contains:
        score (0.01–0.99), breakdown dict, feedback string, passed bool.
    - Determinism requirement: same inputs always yield same score.

4.6 Baseline Policy (baseline/policy.py — 7 KB)

    - Rule-based: inspects observation, applies heuristics.
    - Flow per difficulty:
        Easy  : classify → respond → resolve
        Medium: classify → request_info → respond → resolve
        Hard  : classify → respond → escalate
    - run_baseline.py orchestrates env.reset → policy.act → env.step
      loop for seeds [42, 123, 456], writing results.json.

5. API SPECIFICATION
================================================================================

5.1 REST Endpoints

    GET  /health
         Response: { "status": "healthy", "environment": "SupportEnv" }

    GET  /tasks
         Response: { "tasks": [ { task_id, name, difficulty,
                     description, max_steps, action_schema } ] }

    POST /api/reset
         Body:     { session_id?, seed?, task_id?, difficulty? }
         Response: { session_id, observation, done, reward }

    POST /api/step
         Body:     { session_id, action_type, content, confidence? }
         Response: { observation, done, reward }

    GET  /api/state/{session_id}
         Response: SupportState (JSON)

    POST /grader
         Body:     { session_id }
         Response: { score, breakdown, feedback, passed }

    GET  /baseline
         Response: { baseline_results: { easy, medium, hard },
                     summary: { easy_score, medium_score, hard_score,
                                average_score } }

5.2 WebSocket Endpoint

    WS  /ws
         Follows OpenEnv WebSocket protocol:
           Client sends { type: "reset", ... }  → receives Observation
           Client sends { type: "step", action: {...} } → receives Observation
           Server sends done=true when episode ends.

5.3 Gradio UI

    GET  /web
         Mounted via gr.mount_gradio_app(app, demo, path="/web").
         Interactive interface for manual ticket exploration.

6. DATA MODELS (Pydantic v2)
================================================================================

6.1 SupportAction (extends Action)
    Fields:
      action_type : Literal["classify","respond","escalate",
                            "request_info","resolve"]
      content     : str
      confidence  : Optional[float]  (0.0–1.0)

6.2 SupportObservation (extends Observation)
    Fields:
      ticket_id, ticket_text, ticket_subject, customer_name   : str
      interaction_history  : List[Dict[str, str]]
      customer_sentiment   : float  [-1.0, 1.0]
      current_classification : Optional[str]
      is_classified, is_escalated : bool
      task_difficulty       : Literal["easy","medium","hard"]
      steps_remaining, max_steps : int
      message               : str
      available_actions     : List[str]
    Inherited:
      done   : bool
      reward : Optional[float]

6.3 SupportState (extends State)
    Fields:
      target_category, target_resolution : str
      requires_escalation                : bool
      task_id, task_difficulty            : str
      max_steps                          : int
      classification_correct             : bool
      response_quality_score             : float
      escalation_correct                 : bool
      resolved                           : bool
      total_reward                       : float
    Inherited:
      episode_id : Optional[str]
      step_count : int

11. SECURITY CONSIDERATIONS
================================================================================

11.1 API Authentication
     - Optional API_SECRET_KEY for bearer-token auth.
     - When set, all endpoints require Authorization header.

11.2 CORS
     - Default: allow all origins (*).
     - Production: set CORS_ORIGINS to specific domains.

11.3 Rate Limiting
     - Disabled by default.
     - Enable via RATE_LIMIT_ENABLED=True.
     - Configurable window (default 60 s) and limit (default 100 req).

11.4 Input Validation
     - All inputs validated through Pydantic v2 models.
     - action_type restricted via Literal type.
     - confidence bounded [0.0, 1.0] via Field(ge=0.0, le=1.0).
     - customer_sentiment bounded [-1.0, 1.0].

12. PERFORMANCE & SCALABILITY
================================================================================

12.1 Concurrency
     - SupportEnvironment.SUPPORTS_CONCURRENT_SESSIONS = True
     - In-memory session store (dict) for HTTP endpoints.
     - MAX_CONCURRENT_ENVS = 100 (configurable).

12.2 Latency Budget (single episode)
     Operation        Target
     ──────────────   ──────
     reset()          < 10 ms
     step()           < 20 ms
     grade_episode()  < 30 ms
     Full episode     < 500 ms

12.3 Memory
     - Each session: ~50 KB (ticket + history + state).
     - 100 sessions: ~5 MB total.
     - No persistent storage required.
